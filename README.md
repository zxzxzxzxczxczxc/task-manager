# Task Manager Lite

## 1. Назначение

Одностраничное веб-приложение для управления личными задачами.

## 2. Область реализации

- Формат: клиентское SPA без бэкенда.
- Хранение данных: `localStorage` браузера.
- Запуск: через `node start.js` (открытие `index.html`).
- Сетевые запросы и серверные зависимости: не используются.

## 3. Функциональные требования

### 3.1 Управление задачами

- Пользователь может создать задачу.
- Пользователь может редактировать задачу.
- Пользователь может удалить задачу.
- Для задачи обязательное поле: `title`.
- Для задачи необязательные поля: `description`, `deadline`, `tags`.

### 3.2 Теги

- Пользователь может создать новый тег.
- Пользователь может назначать теги задаче при создании.
- Пользователь может изменять набор тегов у задачи при редактировании.
- Теги должны храниться отдельно и переиспользоваться между задачами.

### 3.3 Поиск

- Поиск выполняется в реальном времени при вводе.
- Поиск должен учитывать:
  - название задачи (`title`);
  - описание (`description`);
  - теги (`tags`);
  - строку дедлайна (`deadline`).

### 3.4 Дедлайны

- Задача может иметь дедлайн в формате `YYYY-MM-DD`.
- Просроченная задача определяется как `deadline < текущая_дата`.
- Просроченные задачи должны иметь визуальную подсветку.

### 3.5 Тема оформления

- Должны поддерживаться 2 темы: `light` и `dark`.
- Переключение темы выполняется кнопкой в интерфейсе.
- Выбранная тема сохраняется между сессиями.

### 3.6 Сохранение между сессиями

- Должны сохраняться:
  - список задач;
  - список тегов;
  - активная тема.
- Данные должны восстанавливаться при повторном открытии приложения.

## 4. Требования к интерфейсу

- Наличие формы добавления задачи.
- Наличие поля поиска.
- Наличие списка задач с действиями `Изменить` и `Удалить`.
- Наличие UI для создания тегов и выбора тегов.
- Наличие модального окна редактирования задачи.

## 5. Модель данных

### 5.1 Задача

```json
{
  "id": "string (UUID)",
  "title": "string",
  "description": "string",
  "deadline": "string (YYYY-MM-DD | empty)",
  "tags": ["string"],
  "createdAt": "number (timestamp)"
}
```

### 5.2 Ключи localStorage

- `task-manager.tasks.v1`: `Task[]`
- `task-manager.tags.v1`: `string[]`
- `task-manager.theme.v1`: `"light" | "dark"`

## 6. Нефункциональные требования

- Минимальная архитектурная сложность.
- Отсутствие внешних runtime-зависимостей.
- Поддержка современных браузеров (Chrome, Edge, Firefox, Safari).
- Интерфейс должен быть адаптивным (desktop/mobile).

## 7. Запуск

```bash
npm start
```

Команда запускает `start.js`, который открывает `index.html` в браузере.

## 8. Критерии приёмки

- Все требования из раздела 3 реализованы.
- После перезапуска браузера данные задач/тегов и тема сохраняются.
- Поиск корректно фильтрует задачи.
- Просроченные задачи визуально отличаются от остальных.
- Приложение запускается через `npm start` без установки дополнительных пакетов.
